## 第一部分 浪潮之巅：为什么 Vibe Coding 是必然

### 第1章 时代背景：软件吞噬世界，AI 重塑软件
- **1.1 从信息化到智能化**：软件已渗透一切，AI 让软件有了“大脑”
- **1.2 算力、数据、算法的螺旋上升**：三要素如何共同催生大模型能力
- **1.3 软件正在成为“第二语言”**：表达想法，不再依赖特定技术方言
- **1.4 创造者瓶颈的消解**：成本归零，唯一剩余的门槛是“意愿”
- **Vibe 练习**：让 AI 为你解释一个你完全陌生的技术概念，感受“对话即学习”

---

### 第2章 生产关系的演变：从手工作坊到智能工厂
- **2.1 开发者群体的演化**：精英俱乐部 → 开源社区 → 全民创造者
- **2.2 开源运动的启示**：协作与复用的力量，如何让个人站在巨人肩上
- **2.3 生产者与消费者的边界溶解**：“想法即交付”，用户即共建者
- **2.4 新分工体系：人类与 AI 的契约**  
  - 人类负责：意图、判断、设计、责任  
  - AI 负责：实现、验证、重复劳动
- **Vibe 日志**：展示一段传统需求文档 vs 一段 Vibe 对话，对比信息密度

### 第3章 生产工具的演变：编程范式的四次解放
- **3.1 机器码与汇编**：直接与硬件对话，只有极少数人能掌握
- **3.2 高级语言与 IDE**：抽象的第一次攀升，释放了专业开发者
- **3.3 框架、低代码与云原生**：从“写”到“组装”，但仍有不可逾越的屏障
- **3.4 自然语言作为编程语言**：Vibe Coding 的终极抽象，解放所有人
- **图示**：四次解放对应的创造者群体数量变化

### 第4章 传统开发生命周期回顾
- **4.1 经典瀑布模型**：需求→设计→开发→测试→部署，单向且脆弱
- **4.2 敏捷与 DevOps 的改进**：迭代加速，但人的沟通成本仍然高昂
- **4.3 每个阶段的“翻译损耗”**  
  - 需求到设计：信息丢失  
  - 设计到代码：实现偏差  
  - 开发到测试：认知错位  
  - 测试到部署：环境差异
- **4.4 人类精力分配真相**：编码其实只占 20%，沟通与等待占大头
- **Vibe 练习**：用 AI 画一张传统生命周期痛点地图

## 第二部分 认知重塑：什么是 Vibe Coding

### 第6章 Vibe Coding 的“原始”要素：Token 即流量
- **6.1 智能手机的启示**：不联网的“砖头”与全能的终端
- **6.2 Token：AI 世界的信息原子**  
  - 什么是 Token（词元），文本如何被切分  
  - Token 是模型理解与生成的唯一交换媒介
- **6.3 “Token 即流量”**：每次 Vibe 都在消耗“算力话费”  
  - 输入 Token（上传）与输出 Token（下载）  
  - 计费与配额的底层逻辑
- **6.4 上下文窗口：你的“实时流量套餐”**  
  - 上下文窗口容量解释  
  - 对话越长，累计 Token 越接近上限，模型“失忆”的风险  
  - 管理上下文就是管理成本
- **6.5 Vibe Coding 的成本思维：意图的性价比**  
  - 高性价比 Vibe：精准描述 + 最小上下文  
  - 挥霍型 Vibe：模糊需求 + 反复试错  
  - 培养“信息密度”直觉
- **Vibe 练习**：用同一个需求，写一段 100 Token 的提示和一段 500 Token 的提示，对比 AI 输出质量

---

### 第7章 Vibe Coding 核心概念
- **7.1 定义与内涵**：以意图和氛围驱动，通过对话迭代式构建软件的范式
- **7.2 四大核心要素**  
  - 意图表达（Prompt/描述）  
  - 上下文管理（贴代码、历史、环境）  
  - AI 执行（生成、修改、提问）  
  - 即时反馈（运行、报错、视觉确认）
- **7.3 “Vibe”的本质**：目标感、流畅感、信任感的融合  
  - 什么时候感觉“Vibe 对了”？  
  - 什么时候感觉“Vibe 断了”？
- **7.4 与低代码/无代码的彻底区别**  
  - 低代码：拖拽组装受限组件，天花板明显  
  - Vibe Coding：生成任意代码，无平台锁定，天花板=大模型能力上限
- **7.5 Vibe Coding 的适用范围**：脚本、Web 应用、自动化、AI 智能体、乃至硬件编程
- **Vibe 日志**：一段“好 Vibe”对话实录与一段“坏 Vibe”对话实录，逐轮分析

---

### 第8章 从 Vibe Coding 到 Vibe Learning
- **8.1 学习模式的革命**：先跑起来，再理解原理
- **8.2 在构建中学习**：AI 是 24 小时在线的导师与结对搭档
- **8.3 “不必弄懂一切”的新契约**  
  - 抽象层的再提升：何时信任 AI，何时深入底层  
  - 学习的目标从“记忆语法”转为“培养判断力”
- **8.4 构建个人学习闭环**  
  提问 → 生成 → 阅读（真的去读代码）→ 尝试修改 → 理解
- **8.5 培养 Vibe 直觉**：如何识别 AI 的“胡说”，如何在复杂需求中保持主线
- **Vibe 练习**：选一个你完全没学过的编程概念，让 AI 写一个可运行的小例子教你

## 第三部分 引擎之下：大模型与工具生态

### 第9章 大模型：Vibe Coding 的认知引擎
- **9.1 模型如何理解代码**

  - 代码生成、推理、长上下文能力的差异  
  - 如何根据任务选模型
- **9.3 提示工程即“氛围设定”**  
  - 系统提示词就是给 AI 的角色与规则  
  - 少样本提示、思维链、角色扮演在 Vibe Coding 中的应用
- **9.4 模型的能力边界**  
  - 幻觉的产生与缓解  
  - 推理深度的物理限制  
  - 遇到数学/逻辑硬伤时的策略
- **9.5 未来趋势**：智能体自主循环、多模态理解、本地小模型
- **Vibe 练习**：让同一个需求分别跑在 Claude 和 GPT 上，对比输出差异

### 第10章 Vibe Coding 工具生态全景
- **10.1 工具分层模型**  
  - L1 终端代理（最强控制，需适应 CLI）  
  - L2 IDE 插件（融合日常编码流）  
  - L3 全栈平台（零配置，浏览器内完成）
- **10.2 终端原生：Claude Code / Gemini CLI**  
  - 工作原理：读取文件系统、运行命令、修改代码  
  - 与传统 CLI 工具的区别  
  - 适合的场景与人群
- **10.3 IDE 集成：GitHub Copilot / Cursor / Windsurf**  
  - 内联补全 vs 对话式侧边栏  
  - Cursor 的“Apply”模式与代码修改流程  
  - 如何与 Claude Code 配合使用
- **10.4 全栈 Vibe 平台：Replit Agent / v0 / bolt.new**  
  - 从一句话想法到上线应用的“魔法”  
  - 各自的局限性（控制力、定制化）  
  - 适合原型验证与快速实验
- **10.5 选择你的“好搭档”**：从入门到精通的工具组合推荐路径
- **Vibe 日志**：用同一句话在不同工具中创建应用，记录体验差异
