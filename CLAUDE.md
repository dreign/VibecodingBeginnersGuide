# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

《Vibe Coding 从入门到精通：AI 原生开发实战指南》— A Chinese-language book about Vibe Coding (AI-native development). The book targets creators, independent developers, and traditional programmers transitioning to AI-assisted development. It covers the paradigm shift from traditional software engineering to AI-native "Vibe Coding," with a focus on the "一人公司" (One-Person Company) concept.

## Book Structure (Six Parts)

- **Part 1** (Ch 1-5): Why Vibe Coding is inevitable — historical context, production relations, tool evolution, traditional lifecycle, one-person company
- **Part 2** (Ch 6-8): What is Vibe Coding — tokens, core concepts, Vibe Learning
- **Part 3** (Ch 9-10): LLMs & tool ecosystem — how models work, tool landscape
- **Part 4** (Ch 11-12): Hands-on practice — environment setup, first application
- **Part 5** (Ch 13-15): Mastery — advanced projects, open-source libraries, debugging
- **Part 6** (Appendices A-C): Quick reference, knowledge wiki, learning resources

Each book chapter should be created as a separate markdown file as writing progresses.

## Writing Conventions

- **Primary language**: Simplified Chinese (简体中文)
- **File naming**: Chinese descriptive names for book-related files
- **Writing style**: Analogy-driven, practical, with "Vibe 练习" (exercises) and "Vibe 日志" (logs) in each chapter
- **Edition note**: Include version/disclaimer about tool versions since AI tools evolve rapidly
- When writing new chapters, follow the outline in `写作大纲.md` for structure and coverage

## Content Guidelines

- Maintain "认知先行，再给武器" (cognition first, then tools) principle
- Use analogies (not raw terminology) for abstract concepts
- Include real dialogue snippets showing good vs bad Vibe interactions
- Every chapter should have a "Vibe 练习" section with actionable AI conversation tasks
- Visual diagrams referenced in the outline should be noted for future creation
- Tool-specific content should be written as "原理不变，工具可替换" (principles不变, tools replaceable)
